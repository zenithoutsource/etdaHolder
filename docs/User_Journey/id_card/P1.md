# P1: Wallet Provisioning & PID VC Issuance
(การเตรียมพร้อม Wallet และการออกเอกสาร PID หลัก)

## Lanes (actors / systems)

| Lane | Role |
|---|---|
| VCGA | Governance authority (lane present, no steps in this diagram) |
| PID Issuer | Verifies the Holder at high LoA and issues the foundational PID VC |

## Flow steps

1. **[Holder]** Start process (เริ่มกระบวนการ)
2. **[Holder]** Download the Digital Wallet (ดาวน์โหลด Digital Wallet)
3. **[OS (iOS/Android)]** Install the Digital Wallet (ติดตั้ง Digital Wallet)
4. **[Holder]** Open the Digital Wallet for the first time (เปิด Digital Wallet ครั้งแรก)
5. **[OS (iOS/Android)]** Set PIN/Biometric — see Figure 1 (ตั้งรหัส PIN / Biometric, รูปที่ 1)
6. **[Digital Wallet]** App ready for use (Operational) — see Figure 2 (แอปพร้อมใช้งาน (Operational), รูปที่ 2)
7. **[Holder]** Request the PID VC document — see Figures 3, 4, 5 (ร้องขอเอกสาร PID VC, รูปที่ 3/4/5)
8. **[Digital Wallet]** Carry out the PID VC request process (ดำเนินกระบวนการขอ PID VC)
9. **[PID Issuer]** Receive the request and have the Holder verify identity at high LoA (LoA High) — see Figure 6 (รับคำร้อง และให้ Holder ยืนยันตัวตนระดับสูง (LoA High), รูปที่ 6)
10. **[Trust Registry]** Confirm the app is properly registered and trustworthy — see Figure 7 (ยืนยันว่าแอปได้ขึ้นทะเบียนถูกต้องและเชื่อถือได้, รูปที่ 7)
11. **[PID Issuer]** Create and digitally sign the PID VC document (สร้างและลงนามเอกสาร PID VC)
12. **[VC Status Registry]** Record the document status as Active (พร้อมใช้งาน) (บันทึกสถานะเอกสารว่าพร้อมใช้งาน (Active))
13. **[Audit Trail]** Record the issuance history — see Figure 8 (บันทึกประวัติการออกเอกสาร, รูปที่ 8)
14. **[PID Issuer]** Deliver the PID VC — see Figure 9 (ส่งมอบ PID VC, รูปที่ 9)
15. **[Digital Wallet]** Store the PID VC document — see Figure 10 (จัดเก็บเอกสาร PID VC, รูปที่ 10)
16. **[Holder]** Acknowledge the result of the PID VC issuance (รับทราบผลการออกเอกสาร PID VC)
17. **[Digital Wallet]** App is legally ready for use (Valid) — see Figure 11 (แอปพร้อมใช้งานตามกฎหมาย (Valid), รูปที่ 11)
18. **[Digital Wallet]** End process (สิ้นสุดกระบวนการ)

## Summary

P1 is the bootstrap journey before any other ID-Card flow can run: the Holder downloads and installs the Wallet (handled by the OS), sets up a PIN/Biometric to reach an "Operational" state, then requests the foundational Personal Identification Data (PID) credential. The PID Issuer requires high-Level-of-Assurance (LoA High) identity verification before issuing — and the Trust Registry independently confirms the Wallet app itself is registered and trustworthy. Once the PID VC is created, signed, marked Active in the VC Status Registry, and logged to the Audit Trail, it is delivered to and stored in the Wallet, which then transitions to a legally "Valid" state — the prerequisite identity anchor that P2 (standard VC issuance) depends on.
