# P0: Ecosystem Governance & Trust Registry
(การกำกับดูแลและขึ้นทะเบียนความน่าเชื่อถือ)

Source: `P0-Transcript - Swimlane Diagram v.1.xlsx`

## Lanes (actors / systems)

| Lane | Role |
|---|---|
| VCGA | Governance authority — registers/approves issuers & verifiers, issues access certificates |
| Issuer / Verifier | Entity requesting registration into the ecosystem |
| Holder | (lane present, no steps in this diagram) |

## Flow steps

1. **[Issuer/Verifier]** Start process (เริ่มกระบวนการ)
2. **[VCGA]** Prepare system: generate signing key (`did:web`); prepare Certificate Signing Request (CSR)
   (เตรียมความพร้อมระบบ - สร้างกุญแจเข้ารหัส (did:web) - คำขอลงนามใบรับรอง (CSR))
3. **[Issuer/Verifier]** Submit registration request (ยื่นคำขอขึ้นทะเบียน)
4. **[VCGA]** Verify legal & technical qualifications (ตรวจสอบคุณสมบัติทางกฎหมาย และ ทางเทคนิค)
5. **[VCGA]** Approve registration — see Figure 1 (อนุมัติการขึ้นทะเบียน, รูปที่ 1)
6. **[Trust Registry]** Record data and publish trust status (บันทึกข้อมูล และประกาศความน่าเชื่อถือ)
7. **[Schema Registry]** Record schema/template structure (บันทึกโครงสร้าง และแม่แบบเอกสาร)
8. **[Audit Trail]** Record registration history (บันทึกประวัติการขึ้นทะเบียน)
9. **[VCGA]** Issue Access Certificate in standard X.509 format (ออกใบรับรอง (Access Certificate) ในรูปแบบมาตรฐาน X.509)
10. **[Issuer/Verifier]** Receive registration result and Access Certificate (รับแจ้งผลการขึ้นทะเบียน - รับใบรับรอง (Access Certificate))
11. **[Issuer/Verifier]** End process (สิ้นสุดกระบวนการ)

## Summary

P0 covers ecosystem onboarding/governance: an Issuer/Verifier prepares its keys (did:web) and a CSR, submits a registration request, VCGA vets it legally/technically and approves it, the Trust Registry + Schema Registry + Audit Trail each persist their respective records, and VCGA issues an X.509 Access Certificate back to the entity — completing the trust-bootstrap step that precedes any credential issuance/verification activity.
