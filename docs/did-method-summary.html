การทำงานจริงในโปรเจกต์นี้
1
เปิดแอปครั้งแรก — สร้าง Holder DID
แอปสร้าง P-256 Keypair ใน Hardware Secure Enclave ของอุปกรณ์ จากนั้นคำนวณ did:key จาก Public Key โดยอัตโนมัติ — ไม่ต้องเชื่อมต่อ Internet ไม่ต้องใช้ Server ใดๆ DID นี้คือ ตัวตนของผู้ถือ Credential
2
สแกน QR Credential Offer
แอปอ่าน URI จาก QR Code แล้วติดต่อ Issuer Server โดยตรง ดึง Metadata ของ Issuer ซึ่งประกอบด้วย did:web ของ Issuer — ใช้ยืนยันว่าเป็น Issuer ที่ถูกต้องตามมาตรฐาน OID4VCI 1.0
3
สร้างและส่ง Proof of Possession JWT
แอปสร้าง JWT และ Sign ด้วย Hardware Key — ระบุ did:key ของ Holder เป็น iss ใน JWT ทุกครั้งที่ Sign จะมีการยืนยัน Biometric ที่ระดับ Hardware นี่คือหลักฐานว่าผู้ขอ Credential เป็นเจ้าของ Key จริง
4
Issuer ตรวจสอบและออก Credential
Issuer ตรวจสอบ Proof JWT แล้วออก JWT VC ที่ Sign ด้วย Private Key ของ did:web ของ Issuer ผู้ที่ได้รับ Credential ในอนาคตสามารถตรวจสอบลายเซ็นของ Issuer ผ่าน DID Document ได้
5
บันทึก Credential ลงอุปกรณ์
Credential ถูกเข้ารหัสและบันทึกใน Encrypted MMKV Storage พร้อม Sync ไปยัง Backend พร้อม did:key ของ Holder เพื่อใช้อ้างอิงในอนาคต
did:key คือตัวตนของ ผู้ถือ Credential (Holder) — สร้างจาก Hardware Key ในอุปกรณ์ ไม่ต้องพึ่งพา Server ใดๆ  ·  did:web คือตัวตนของ ผู้ออก Credential (Issuer) — ผูกกับ Domain Name และ HTTPS Infrastructure ของ Issuer
